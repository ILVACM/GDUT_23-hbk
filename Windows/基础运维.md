## 网络连接冗余命名清除

### 现象原因

Windows 系统在每次检测到网络环境变化（如更换网卡、重装驱动、网络配置变更）时，会在注册表中创建新的网络配置文件记录。长期累积会导致：

- 网络名称出现"网络 2"、"网络 3"等冗余编号
- 注册表中存储大量无用的历史网络配置文件

### 注册表路径

```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NetworkList\Profiles
```

每个子项以 `{GUID}` 格式命名，`ProfileName` 值存储网络显示名称。

### 处理方法

使用自动化脚本 `Windows_CleanNetRecord.py` 进行清理：

```powershell
# 以管理员身份运行
python Windows_CleanNetRecord.py
```

**脚本功能**：
- 自动枚举所有网络配置文件
- 支持正向/反向选择模式
- 删除前自动备份注册表项
- 完整的操作日志记录

### 注意事项

1. **必须以管理员权限运行**
2. 建议操作前创建系统还原点
3. 删除后重启计算机使更改生效
4. 备份文件位于脚本目录的 `NetworkBackup` 文件夹
